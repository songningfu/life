# 事件编辑器使用说明

## 简介

这是一个为《大学四年（LIFE）》游戏开发的可视化事件编辑器，可以方便地增加、删除、修改游戏事件，无需手动编辑 JSON 文件。

## 启动编辑器

1. 在 Godot 编辑器中打开项目
2. 在场景树中打开 `scenes/EventEditor.tscn`
3. 点击运行当前场景（F6）或主场景运行按钮

## 界面布局

编辑器分为左右两个面板：

### 左侧面板 - 事件列表
- **搜索栏**：输入事件ID或标题进行搜索
- **类型筛选**：可选择查看全部/story/daily类型事件
- **事件列表**：显示所有符合条件的事件
- **新建事件**：创建一个新的空白事件
- **删除事件**：删除当前选中的事件
- **保存到文件**：将所有修改保存到 `data/events.json`

### 右侧面板 - 事件编辑器
- **基础信息**：事件ID、标题、类型、描述
- **触发条件**：年级范围、需要标签、排除标签
- **选项列表**：事件的所有选择分支
- **应用修改**：保存当前事件的修改

## 使用流程

### 1. 查看和编辑现有事件

1. 在左侧事件列表中点击要编辑的事件
2. 右侧编辑器会自动加载该事件的所有信息
3. 修改任何字段（ID、标题、描述、条件等）
4. 编辑选项内容和效果
5. 点击"应用修改"按钮保存更改
6. 点击"保存到文件"将更改写入JSON文件

### 2. 创建新事件

1. 点击左侧"新建事件"按钮
2. 系统会创建一个带有默认值的新事件
3. 修改事件ID（建议使用有意义的命名，如 `y1s1_library`）
4. 填写标题和描述
5. 设置触发条件（年级、标签等）
6. 添加选项（见下文）
7. 点击"应用修改"
8. 点击"保存到文件"

### 3. 删除事件

1. 在左侧列表中选择要删除的事件
2. 点击"删除事件"按钮
3. 事件会立即从内存中删除
4. 点击"保存到文件"使删除生效

## 选项编辑器详解

每个事件可以有多个选项，每个选项包含：

### 选项文本
玩家看到的选择内容，例如："去图书馆学习"

### 属性效果
选择该选项后对玩家属性的影响：
- **学习**：影响 study_points（-50 到 +50）
- **社交**：影响 social（-50 到 +50）
- **能力**：影响 ability（-50 到 +50）
- **生活费**：影响 living_money（-500 到 +500，单位：元）
- **心理**：影响 mental（-50 到 +50）
- **健康**：影响 health（-50 到 +50）

> 提示：值为0的属性不会被保存到JSON中

### 标签管理
- **添加标签**：选择该选项后添加的标签，用逗号分隔
  - 例如：`debate_club,first_competition`
- **移除标签**：选择该选项后移除的标签，用逗号分隔
  - 例如：`no_club,hesitant`

### 效果提示
可选字段，用于向玩家展示选择后的效果提示文本
- 例如："你感到精疲力竭，但学到了很多"

### 添加和删除选项
- 点击"添加选项"按钮创建新选项
- 点击选项右上角的"删除"按钮移除该选项

## 事件字段说明

### 基础信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 事件ID | 唯一标识符，建议使用下划线命名 | `y1s1_first_class` |
| 标题 | 事件标题，显示在游戏中 | "第一节专业课" |
| 类型 | story（主线）或 daily（日常） | `story` |
| 描述 | 事件的详细描述文本 | "你走进教室..." |

### 触发条件

| 字段 | 说明 | 示例 |
|------|------|------|
| 年级范围 | 事件可触发的年级（1-4） | 最小：1，最大：1 |
| 需要标签 | 必须拥有的标签才能触发 | `debate_club,active` |
| 排除标签 | 拥有这些标签则不触发 | `no_club,graduated` |

## 注意事项

### 1. 保存机制
- 编辑器在内存中操作，必须点击"保存到文件"才会写入磁盘
- 建议定期保存，避免数据丢失
- 保存前可以先备份 `data/events.json` 文件

### 2. 事件ID命名规范
- 使用小写字母和下划线
- story事件建议格式：`y{年级}s{学期}_{描述}`
  - 例如：`y1s1_roommate`（大一上学期室友事件）
- daily事件建议格式：`daily_{描述}`
  - 例如：`daily_library_seat`（日常图书馆占座事件）

### 3. 标签使用建议
- 标签用于控制剧情分支和事件触发
- 常用标签类型：
  - 社团：`debate_club`, `tech_club`, `student_union`
  - 关系：`crush`, `in_relationship`, `broke_up`
  - 路线：`want_postgrad`, `want_job`, `want_stable`
  - 成就：`debate_winner`, `first_project`, `competition_exp`

### 4. 属性平衡
- 注意属性效果的平衡性
- 高收益选项通常应伴随代价（如学习+10但健康-5）
- 避免单个选项给予过高的属性值（建议单次不超过±15）

### 5. 测试建议
- 修改事件后在游戏中测试触发条件
- 检查选项效果是否符合预期
- 验证标签逻辑是否正确

## 快捷操作

- **搜索事件**：直接在搜索栏输入关键词
- **快速筛选**：使用类型下拉菜单
- **批量编辑**：可以连续选择多个事件进行编辑，每次修改后点击"应用修改"

## 故障排除

### 问题：编辑器无法加载事件
- 检查 `data/events.json` 文件是否存在
- 检查JSON格式是否正确（可用在线JSON验证工具）

### 问题：保存后游戏中没有变化
- 确认已点击"保存到文件"按钮
- 重新启动游戏场景以加载新数据
- 检查控制台是否有错误信息

### 问题：选项效果不生效
- 确认属性名称正确（study_points, social, ability, living_money, mental, health）
- 检查值是否在有效范围内
- 确认已点击"应用修改"和"保存到文件"
