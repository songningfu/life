# 《大学四年》(LIFE) 完整项目文档

## 项目概述

《大学四年》是一款大学人生模拟游戏，使用 Godot 4.6 + GDScript 2.0 开发。玩家扮演一名大学生，经历四年的学习、社交、恋爱和成长，最终迎来毕业和不同的人生结局。

## 核心设计理念

1. **双速时间系统**：重要日手动操作，普通日自动推进
2. **模块化架构**：所有系统都是可插拔的模块
3. **数据驱动**：游戏内容通过JSON配置，易于扩展
4. **情感弧线**：大一新奇→大二习惯→大三分化→大四告别

## 二、核心框架

### 2.1 模块系统架构

#### GameModule.gd - 模块基类

所有游戏系统（天赋、恋爱、健身等）都继承此类。定义标准接口：

**身份方法：**
- `get_module_id()` - 模块唯一ID
- `get_module_name()` - 模块显示名

**生命周期钩子：**
- `on_new_game()` - 新游戏初始化
- `on_day_start()` - 每天开始
- `on_action_performed()` - 玩家执行行动后
- `on_day_end()` - 每天结束
- `on_week_end()` - 每周结算
- `on_semester_end()` - 学期结算

**数据注入接口：**
- `get_available_actions()` - 提供可用行动
- `get_modifiers()` - 提供属性修正
- `get_daily_passive_effects()` - 提供每日被动效果
- `get_event_injections()` - 提供事件注入
- `get_morning_info()` - 提供晨间信息
- `get_ui_panels()` - 提供手机App面板
- `get_sendable_messages()` - 提供可发送消息
- `get_npc_outgoing_messages()` - 提供NPC主动消息

**序列化：**
- `serialize()` / `deserialize()` - 存档支持

#### ModuleManager.gd - 模块管理器

负责模块的注册、广播、聚合。

**核心方法：**
- `register(module)` - 注册模块
- `get_module(id)` - 获取模块实例
- `broadcast_*()` - 广播生命周期事件
- `collect_*()` - 聚合各模块数据

#### ModLoader.gd - 模组加载器

支持两种模组类型：

1. **代码模组**：用户编写继承GameModule的GDScript
2. **纯数据模组**：仅JSON文件，由DataPackModule加载

### 2.2 核心循环

Game.gd 实现分阶段状态机：

```
MORNING_INFO (晨间信息)
    ↓
SLOT_MORNING (上午时段)
    ↓
SLOT_AFTERNOON (下午时段)
    ↓
SLOT_EVENING (晚上时段)
    ↓
NIGHT_SUMMARY (夜间结算)
    ↓
回到 MORNING_INFO
```

**普通日**：按日程模板自动执行
**重要日**：玩家手动选择每个时段的行动

### 4.1 五位可攻略角色

| 角色 | ID | 认识方式 | 核心冲突 | 感情基调 |
|------|-----|----------|----------|----------|
| 林知意 | lin_zhiyi | 图书馆偶遇 | 她要出国 | 温暖而克制 |
| 苏小晚 | su_xiaowan | 社团招新 | 性格反差 | 热烈而冲突 |
| 陈雨桐 | chen_yutong | 隔壁班（开局认识） | 暗恋不自知 | 细水长流 |
| 周一然 | zhou_yiran | 兼职打工遇到 | 家境差异 | 现实而坚韧 |
| 沈映霜 | shen_yingshuang | 考研自习室 | 同为考研人 | 互相支撑 |

### 4.2 恋爱阶段

```
NOT_MET(未遇见) → MET(认识) → FAMILIAR(熟悉) → AMBIGUOUS(暧昧) → CONFESSION(表白节点) → DATING(交往中) → STABLE(稳定) → [BREAKUP(分手)/REGRET(遗憾)]
```

### 4.3 心动值系统

- **好感度(affinity)**：NPC对你的态度，0~100
- **心动值(spark)**：暧昧浓度，0~100

两者独立但共同决定关系走向：
- MET → FAMILIAR：affinity≥20 且 spark≥15
- FAMILIAR → AMBIGUOUS：affinity≥40 且 spark≥35
- AMBIGUOUS → CONFESSION：affinity≥55 且 spark≥70

**暧昧期衰减**：超过5天没互动，spark每天-1

### 4.4 角色故事线（以林知意为例）

**大一上 [偶遇期]**：
- 图书馆行动25%概率触发初遇
- 第一次互动：帮她捡书签
- 加微信时她犹豫了一下

**大一下 [熟悉期]**：
- 开始偶尔一起占座
- 分享英语学习资料
- 考试周一起待到闭馆

**大二 [暧昧期]**：
- 固定"约定座位"
- 得知她在准备托福
- 一次心情不好，校园走了很久

**大二末 [分支点]**：
- 她说决定申请出国
- 选项A: 支持她→"克制线"
- 选项B: 表白→"挽留线"
- 选项C: 疏远→"遗憾线"

**大三~大四 [结局分化]**：
- 克制线：毕业前她在机场发了很长的微信
- 挽留线：在一起了但异地压力大
- 遗憾线：毕业典礼远远对视一眼

## 五、手机系统

### 5.1 定位

手机是**不占用行动时段的"免费行动"**，但有使用限制。

### 5.2 使用时机

- 晨间阶段（看完信息后）
- 每个时段行动选择前
- 夜间结算"睡前刷手机"环节

### 5.3 主动发消息

**每天上限3条**，被动回复不限。

消息选项根据关系阶段、游戏阶段动态生成，由各模块注入。

### 5.4 NPC回复机制

- **林知意**：慢，半天到一天，但走心
- **苏小晚**：秒回，语音条轰炸
- **周一然**：有时不回，真的在忙

**等待焦虑**：发了消息没收到回复 → mental临时-1 → 收到回复后反弹+2

### 5.5 朋友圈

NPC不定期发朋友圈，玩家可点赞/评论。核心价值是被动了解NPC近况。

## 九、情感弧线设计

### 大一：新奇→兴奋→迷茫
- 行动选项多但玩家不知道哪个好
- NPC关系快速推进
- 大量"第一次"事件

### 大二：习惯→舒适→倦怠
- 日程模板自动推进变多
- 事件变少但更深入
- 开始出现"选择焦虑"

### 大三：分化→压力→成长
- 强制分流决策（考研/就业/出国）
- NPC路线分化
- 有人开始"消失"

### 大四：告别→不舍→释然
- 行动选项逐渐减少
- NPC陆续离开
- 大量回忆性事件
- 毕业倒计时

## 十、技术规范

### 10.1 命名规范

- **GDScript**：snake_case（`health_changed`）
- **信号**：snake_case
- **类名**：PascalCase（`GameModule`）
- **常量**：UPPER_SNAKE_CASE

### 10.2 类型安全

- 所有变量、参数、返回类型必须显式类型化
- 使用类型化数组：`Array[EnemyData]`
- 使用 `@onready` 类型化节点引用

### 10.3 信号规范

```gdscript
# GDScript
signal health_changed(new_health: float)
signal died

# 连接
health_component.died.connect(_on_died)
```

### 10.4 节点组合

优先使用组合而非继承：
```gdscript
# 好：通过子节点组合
@onready var health: HealthComponent = $HealthComponent

# 不好：深层继承
class_name PlayerWithHealth extends CharacterWithHealth
```
