## 0.1 本次功能更新（v5.1 学业 + 生活费双轨重构）

本次已完成以下核心改动：

- 学业轨道：
  - `study_points`（0~100）作为日常学习状态
  - `gpa`（0.00~4.00）改为学期结算累计值
  - 新增 `semester_records` 与 `academic_warning_count`
  - 学期结算使用分段函数 `_map_study_to_semester_gpa()`
  - 连续三学期绩点过低触发劝退
- 结算后学习点向 65 回归 30%
- 生活费轨道：
  - `living_money`（人民币元）替代旧 `money`
  - 新增 `monthly_allowance`、`daily_base_expense`
  - 每月 1 日自动到账，每日按阶段自动扣费
  - 余额 ≤ 0 时每日额外心理惩罚，不再因金钱直接退学

- 数据与界面适配：
  - `events.json`：`effects.money -> effects.living_money`
  - `events.json`：`conditions.money -> conditions.living_money` 并完成阈值换算
  - `flavor_texts.json`：`conditions.money -> conditions.living_money`
  - `Game.tscn`：右侧改为四个进度条 + GPA/生活费独立展示
  - `SaveManager.gd`：旧存档迁移（`living_money = money * 30 + 500`）
- 存档兼容：新增保存 `study_points`、`semester_records`、`academic_warning_count`、`living_money`，并兼容旧档

## 2. 当前系统快照（2026-03-24）

## 2.1 事件与内容规模

- 事件总量：`54`
  - `story`: `24`
  - `daily`: `30`
- flavor 文本条目：`45`
- 主要 NPC：`9`
- 存档槽位：`3`

## 2.2 核心模块职责

- `Game.gd`：主循环、日历、事件触发、结算、结局、存档对接
- `MainMenu.gd`：新开局/读档入口、角色创建、槽位管理
- `SaveManager.gd`：存档与元数据落盘/读取
- `NamePool.gd`：NPC 姓名池、昵称生成、序列化
- `RelationshipManager.gd`：好感度与关系等级
- `WechatSystem.gd`：模板消息、回复分支、效果结算
- `PhoneSystem.gd`：手机 UI 与 App 路由
- `AudioManager.gd`：BGM 播放与淡入淡出

## 2.3 数据文件角色

- `data/events.json`：事件定义源（触发条件 + 选项效果）
- `data/flavor_texts.json`：无事件日补充叙事文本
